项目概览

StackChan 是一款由 M5Stack 与用户社区联合打造的AI 桌面机器人。主控采用 ESP32-S3（240MHz 双核，16MB Flash + 8MB PSRAM），配备 2.0 寸触摸屏、摄像头、IMU 传感器、麦克风阵列、扬声器、RGB LED 及双伺服电机，支持 Wi-Fi / BLE / ESP-NOW 多种无线通信方式。

本仓库包含四大部分的开源资源：

详细耦合说明

App ↔ 服务器（高耦合）

REST API（Dio HTTP）用于设备注册、绑定、舞蹈同步；WebSocket 自定义二进制协议用于实时音视频通话与控制信令中继。

固件 ↔ 服务器（高耦合）

固件通过 WebSocket 连接服务器，上传摄像头画面 (JPEG) 和音频 (Opus)，接收 App 转发的控制命令和 AI 语音回复。

遥控器 ↔ 固件（低耦合）

遥控器通过 ESP-NOW 广播 8 字节数据包直接控制固件的舵机，不经过服务器；固件独立解析。

App ↔ 固件（BLE / WiFi 直连）

App 可通过 BLE GATT 或局域网 WebSocket 直连机器人，用于设备配网、舞蹈录制等本地场景，无需服务器介入。

固件 ↔ 小智 AI 平台（外部耦合）

固件的 AI Agent App 通过 WebSocket 连接 xiaozhi.me 平台，获取 LLM 对话能力；服务器也通过 REST API 管理小智平台的设备与智能体。

❌ 无直接耦合

遥控器 ↔ 服务器：遥控器完全独立，不与服务器交互。
App ↔ 遥控器：两者不直接通信。

核心数据流

场景一：AI 对话

App 麦克风 → Opus 编码 → WebSocket → Server 中继 → WebSocket → 固件 → 小智 AI 平台 → TTS → WebSocket → Server 中继 → App 扬声器

场景二：遥控器控制

摇杆/IMU → ESP-NOW 8B 包 → 固件 ESP-NOW App → StackChan::updateMotion() → 伺服电机

场景三：摄像头监控

固件摄像头 → JPEG 编码 → WebSocket → Server 中继 → WebSocket → App → 解码显示

实现要点说明

📟 固件核心机制

mooncake 应用框架

每个功能（AI、Avatar、舞蹈等）是一个独立的 AppAbility，通过 installApp 注册，由 mooncake 事件循环统一调度。App 间通过 HAL 共享硬件资源。

StackChan 引擎

核心类组合了 Motion（舵机）、Avatar（面部渲染）、NeonLight（RGB）和 Modifier 池。Modifier 实现"叠加式"动画行为（如说话时嘴动 + 眨眼 + 呼吸可同时生效）。

关键帧动画

KeyframeSequence 定义面部特征（眼睛/嘴位置、缩放）和舵机角度的关键帧，Timeline 负责播放控制（循环、暂停）。支持从 JSON 远程加载。

HAL 硬件抽象层

所有硬件访问（BLE、WiFi、IMU、舵机、触摸、RTC）通过 Hal 接口统一封装。支持 80+ 种开发板，通过 Kconfig menuconfig 选择。

无线通信栈

三模通信：BLE (NimBLE) 用于配网和本地控制、WebSocket 用于云端中继、ESP-NOW 用于低延迟遥控器。三种协议互不干扰。

Avatar 面部系统

基于 LVGL 绘制。采用"皮肤 (Skin)"模式：默认皮肤包含眼睛、嘴、气泡等元素。Decorator 可叠加瞬态装饰（爱心、冷汗、眩晕等），支持表情切换。

📱 App 核心机制

GetX 状态管理

AppState 单例管理全局状态（设备列表、登录态、连接模式）。响应式编程通过 Rx 变量和 Obx widget 实现。

BLE 引擎

flutter_blue_plus 封装，支持三种连接模式：WiFi 直连（Mode 1）、舞蹈录制（Mode 2）、设备配对（Mode 3）。数据通过特征 (Characteristics) 读写。

WebSocket 二进制协议

自定义协议：1 字节类型 + 4 字节长度 (BE) + 负载。支持 Opus 音频流、JPEG 视频帧、控制命令的实时双向传输。自动重连 + Ping/Pong 保活。

双网架构

同时连接自建服务器（设备管理/舞蹈/社交）和小智 AI 平台（智能体/LLM/TTS）。两个网络通过 XiaoZhi_util 独立管理 token。

音频处理管线

AudioEngineManager 管理 Opus 编解码（16kHz, 单声道, 320 帧）。录音通过原生桥接 (MethodChannel) 启动，JNI/Swift 侧采集 PCM 数据，Dart 侧编码为 Opus 发送。

人脸追踪与 AR

Google ML Kit 检测人脸关键点，映射到 3D 机器人模型（three_js 或原生 SceneKit），实现"Avatar 镜像"效果——机器人面部跟随用户表情。

🖥️ 服务器核心机制

WebSocket 中继引擎

核心是 gorilla/websocket 实现的"房间"模式。维护两个连接池：stackChanClientPool（按 MAC 索引）和 appClientPool。消息通过类型路由（0x01~0x1A）。

双层认证模型

V1 API 使用 RSA-OAEP 加密的 MAC 地址（从 Authorization 头解码）。V2 API 使用 HMAC-SHA256 JWT（有效期 1 年）。WebSocket 通过 Base64(RSA(MAC)) 认证。

双 API 版本

V1 基于 MAC 地址的传统认证，包含完整功能。V2 基于 JWT 的现代认证，提供精简的 REST API。两者共存于同一服务。

外部系统集成

用户认证委托给远程 NodeBB 论坛；设备与智能体管理通过 REST API 操作小智 AI 平台（xiaozhi.me），Token 自动 24h 刷新。

🎮 遥控器核心机制

三模式切换

SETUP（配置频道/ID）→ RUNNING（摇杆控制）→ IMU（体感控制）。BtnA 循环切换，BtnB 切换选择项/按钮状态。模式间通过共享数据结构 (joystick_data_t) 交互。

ESP-NOW 数据包

8 字节固定格式：目标 ID + Yaw(int16) + Pitch(int16) + Speed(int16) + BtnB(bool)。无连接、无确认的广播协议，延迟极低（ms 级）。

死区与滤波

摇杆中心区 ±300 的死区防止抖动。变化量小于阈值（5 或 10）时不上报，避免网络拥塞。摇杆原始值映射到舵机物理角度范围。

3D 姿态显示

IMU 模式下，LCD 显示一个旋转的 3D 线框立方体。通过旋转矩阵计算顶点投影，实时反映设备俯仰/滚转姿态，提供直观的体感控制反馈。
